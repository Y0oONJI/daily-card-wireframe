# 데일리 카드 와이어프레임 — 작동 흐름 설명

이 문서는 `wireframe.html`(배포 링크: https://y0oonji.github.io/daily-card-wireframe/)이 어떻게 동작하는지 설명합니다. 지금 만든 건 **실제 서버 없이 브라우저 안에서만 동작하는 클릭형 목업**이에요. 진짜 서비스를 만들 때는 "브라우저에 저장"이라고 적힌 부분들을 서버/DB로 옮겨야 합니다. 그 부분은 마지막에 따로 정리해뒀어요.

---

## 1. 화면 구성

화면은 총 5개 + 부가 요소 2개로 이루어져 있어요.

| 번호 | 화면 | 내용 |
|---|---|---|
| 1 | 오늘의 질문 | 오늘의 질문 텍스트 + "답변하기" 버튼 |
| 2 | 답변 작성 | 질문 다시 보여주고, 텍스트 입력칸 |
| 3 | 배경 선택 | 컬러/그라디언트/이미지 탭 + 카드 미리보기 |
| 4 | 내 카드 결과 | 완성된 카드 + 다운로드 / 타인 답변 보기 버튼 |
| 5 | 타인 답변 보기 | 다른 사람 답변 1개 무작위 노출 |

부가 요소:
- **완료 모달**: 1번 화면 위에 덮이는 팝업. "오늘 이미 답변했다"는 상태일 때만 나타남.
- **새 질문 토스트**: 화면 위쪽에 잠깐 떴다 사라지는 작은 알림. "날짜가 바뀌어서 새 질문이 왔다"는 걸 알려줌.

---

## 2. 기본 흐름 (오늘 처음 들어왔을 때)

```
1.오늘의 질문 → 2.답변 작성 → 3.배경 선택 → 4.내 카드 결과 → 5.타인 답변 보기
                                                                    ↓
                                                           "처음으로" 누르면 1로 복귀
```

각 단계 동작:
1. **1번**: 오늘 날짜에 해당하는 질문을 보여줌 (질문이 어떻게 정해지는지는 4장에서 설명)
2. **2번**: 사용자가 답을 입력. "제출하기"를 누르면 3번으로 이동 (이 시점엔 아직 저장 안 됨)
3. **3번**: 배경(컬러/그라디언트/이미지)을 고르면 미리보기 카드 배경이 바로 바뀜. "카드 만들기"를 누르는 순간 — **이때 비로소 답변+배경이 저장됨** — 4번으로 이동
4. **4번**: 완성된 카드를 보여줌. 다운로드 버튼은 지금은 목업이라 알림창만 뜸(실제로는 카드를 이미지 파일로 만들어 다운로드해야 함)
5. **5번**: "다른 사람 답변 보기"를 누르면 타인의 답변 1개를 보여줌. "처음으로"를 누르면 1번으로 돌아가는데, 이때 그냥 1번으로 가는 게 아니라 "오늘 이미 답변했는지" 다시 확인하는 절차를 거침 (3장 참고)

---

## 3. 같은 날 다시 들어왔을 때 (재방문)

오늘 이미 답변을 완료한 사용자가 같은 날 다시 들어오면 — 즉, 1번 화면에 진입할 때마다(페이지 새로고침, "처음으로" 클릭 등) — 다음을 확인합니다.

> "오늘 날짜로 저장된 답변 기록이 있는가?"

- **있으면**: 1번 화면 위에 **완료 모달**이 덮어 씌워짐. "오늘 답변 완료! 내일 새 질문을 기다려주세요" 문구와 함께 자기가 만든 카드 미리보기, "카드 다시보기" 버튼이 보임. 이 버튼을 눌러야 4번(내 카드) 화면으로 갈 수 있음. (1번의 "답변하기" 버튼은 모달에 가려져서 누를 수 없음 — 즉 하루에 한 번만 답변 가능)
- **없으면**: 평소처럼 질문 화면이 보임

이 판단은 **사용자 계정이 아니라 "이 브라우저"를 기준**으로 합니다. 로그인이 없는 구조라서, 다른 기기나 다른 브라우저로 들어오면 "오늘 답변 안 한 사람"으로 보입니다.

---

## 4. 날짜가 바뀌었을 때 (다음 날)

### 질문이 어떻게 정해지는가
질문은 미리 정해둔 풀(지금은 예시로 5개) 안에서, **날짜를 기준으로 순환**하며 정해집니다. 즉 "2024년 1월 1일부터 오늘까지 며칠 지났는지"를 계산해서, 그 숫자를 질문 개수(5)로 나눈 나머지로 몇 번째 질문인지 고릅니다. 같은 날엔 항상 같은 질문이 나오고, 날짜가 바뀌면 다음 질문으로 자동으로 넘어갑니다.

> 실제 서비스에서는: "질문 20개를 운영자가 직접 등록"하기로 했으니, 이 순환 로직 대신 "오늘 날짜에 매칭된 질문을 DB에서 가져오기"로 바뀌어야 합니다. 지금 이 목업의 5개 질문은 그 자리를 채운 예시일 뿐입니다.

### 새 질문 도착 토스트
"마지막으로 봤던 날짜"를 따로 기억해두고, 1번 화면에 들어올 때마다 오늘 날짜와 비교합니다. 날짜가 달라졌으면(=하루가 지났으면) 화면 위쪽에 "새로운 질문이 도착했어요" 토스트를 잠깐 띄우고, "마지막으로 본 날짜"를 오늘로 갱신합니다. 같은 날 안에서는 한 번 본 뒤로는 다시 뜨지 않습니다.

---

## 5. 데모용 버튼 (실제 서비스엔 없음)

화면 맨 아래에 작게 있는 두 버튼은 **팀원이 테스트하기 편하도록 임시로 넣은 것**이며, 실제 출시 버전에는 빼야 합니다.

- **⏭ 다음날로 이동**: 실제 자정을 기다리지 않고 "하루가 지난 것처럼" 흉내 냅니다. 이걸 누르면 완료 모달이 사라지고 새 질문 + 토스트가 뜹니다.
- **⟲ 데모 초기화**: 흉내 낸 날짜와 오늘 답변 기록을 모두 지우고 맨 처음 상태로 되돌립니다.

---

## 6. 지금 "브라우저에 저장"되는 것들 (실제 서비스에선 서버로 옮겨야 함)

| 저장 항목 | 지금 (목업) | 실제 서비스에서는 |
|---|---|---|
| 오늘 작성한 답변 + 배경 | 브라우저 localStorage | 서버 DB에 저장, 익명 식별자 단위로 |
| 타인 답변 노출 | 항상 같은 문구 하나로 고정 | DB에서 같은 질문에 달린 답변 중 무작위 1개 조회 |
| 욕설 필터 | 없음 (목업이라 생략) | 제출 시 서버에서 금칙어 검사 후 차단 |
| 질문 목록 | 코드 안에 5개 하드코딩 | 운영자가 등록한 질문 DB에서 날짜로 조회 |
| 카드 이미지 다운로드 | 알림창만 뜸 | 카드 영역을 실제 이미지로 변환해 다운로드 (html2canvas 등) |

## 한눈에 보는 전체 분기도

```
사용자가 1번 화면(=앱 진입)에 들어옴
  │
  ├─ 오늘 답변한 기록이 있는가?
  │     ├─ 있음 → 완료 모달 표시 → [카드 다시보기] → 4번 화면
  │     └─ 없음 → 질문 화면 정상 표시
  │
  ├─ 마지막으로 본 날짜와 오늘이 다른가?
  │     ├─ 다름 → "새로운 질문이 도착했어요" 토스트 표시
  │     └─ 같음 → 아무 알림 없음
  │
  └─ (답변 안 한 경우) 답변하기 → 2 → 3 → 4 → 5 → 처음으로 (위 분기 다시 탐)
```
